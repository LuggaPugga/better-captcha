## Overview

Every Better CAPTCHA component now accepts a `scriptOptions` prop. Use it to control how each provider loads its remote script before rendering the widget.

- `autoLoad` (default `true`): set to `false` if you prefer to inject the provider script yourself and let Better CAPTCHA skip automatic loading.
- `timeout` (default `15000` milliseconds): override the maximum time Better CAPTCHA waits for a script to finish loading before throwing an error.

If you disable automatic loading, make sure the provider script is available globally before you call `render()` or mount the component. Once the script is present, the provider behaves exactly the same as when it is auto-loaded.

## Manual Loading Checklist

- Add the provider script tag to your page before rendering the component.
- Set `scriptOptions.autoLoad` to `false` so Better CAPTCHA skips its built-in loader.
- (Optional) Adjust `scriptOptions.timeout` if you rely on slower networks or custom script hosts.

Once configured, the rest of the integration—including callbacks and lifecycle methods—continues to work the same way across all frameworks.
